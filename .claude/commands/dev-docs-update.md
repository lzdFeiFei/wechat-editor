我们正在接近上下文限制。请更新开发文档以确保上下文重置后能够无缝继续。

## 必需的更新

### 1. 更新活动任务文档
对于 `/dev/active/` 中的每个任务：
- 更新 `[task-name]-context.md`，包含：
  - 当前实现状态
  - 本次会话中做出的关键决策
  - 修改的文件及原因
  - 发现的任何阻塞点或问题
  - 接下来的即时步骤
  - 最后更新时间戳

- 更新 `[task-name]-tasks.md`，包含：
  - 将已完成的任务标记为 ✅
  - 添加发现的任何新任务
  - 更新进行中任务的当前状态
  - 如需要则重新排列优先级

### 2. 捕获会话上下文
包含以下相关信息：
- 解决的复杂问题
- 做出的架构决策
- 发现并修复的棘手 bug
- 发现的集成点
- 使用的测试方法
- 进行的性能优化

### 3. 更新记忆（如适用）
- 在项目记忆/文档中存储任何新的模式或解决方案
- 更新发现的实体关系
- 添加关于系统行为的观察

### 4. 记录未完成的工作
- 接近上下文限制时正在进行的工作
- 任何部分完成功能的确切状态
- 重启时需要运行的命令
- 任何需要永久修复的临时解决方案

### 5. 创建交接笔记
如果切换到新对话：
- 正在编辑的确切文件和行号
- 当前更改的目标
- 任何需要注意的未提交更改
- 验证工作的测试命令
