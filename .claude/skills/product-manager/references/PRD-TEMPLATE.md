## 1. 需求概述

### 1.1 需求背景

**业务背景**
- 当前业务现状
- 面临的问题或挑战
- 市场或竞争环境

**用户场景**
- 用户是谁?
- 用户在什么场景下遇到什么问题?
- 用户的期望是什么?

**需求来源**
- 用户反馈/调研
- 数据分析发现
- 市场机会
- 战略规划
- 老板要求

### 1.2 产品目标

**用户目标**
- 为用户解决什么问题?
- 给用户带来什么价值?
- 提升用户的什么体验?

**商业目标**
- 支持公司哪些战略目标?
- 预期产生的商业价值?
- 对核心指标的影响?（GMV、DAU、留存率等）

**目标用户**
- 主要目标用户画像
- 次要目标用户画像
- 用户规模估算

**本期范围（MVP）**

- 核心功能3

## 模板使用说明

1. **章节选择**：根据项目实际情况,选择性使用模板中的章节,不必全部包含
2. **详略得当**：重要功能详细描述,次要功能简要说明
3. **图文并茂**：适当使用流程图、原型图、架构图等辅助说明
4. **持续更新**：PRD是活文档,随着项目推进持续更新完善
5. **版本管理**：重要变更需更新版本号和更新记录
6. **评审通过**：正式开发前必须经过团队评审并获得批准

## 不同类型PRD的裁剪建议

**小功能PRD**（<5人天开发量）
- 保留：需求概述、功能需求、验收标准
- 可省略：市场分析、竞品分析、详细技术方案

**中等功能PRD**（5-20人天开发量）
- 保留：需求概述、功能需求、非功能需求、项目规划
- 简化：市场分析、竞品分析

**大型项目PRD**（>20人天开发量）
- 使用完整模板
- 可能需要拆分为多个子PRD

**新产品PRD**
- 强化：市场分析、竞品分析、用户研究、商业模式
- 完整使用所有章节

**功能优化PRD**
- 强化：现状分析、问题定位、数据支撑
- 可简化：市场分析章节
